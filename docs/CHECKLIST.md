## 기능 요구사항

- 각 자동차의 기종을 선택하고, 이름을 부여할 수 있다. 기종과 이름을 `-`로 연결하여 입력한다
    - `BIKE`, `CAR`, `FERRARI`, `TAXI`, `BUS` 의 기종이 있다.
    - 이름은 5자 이하만 가능하다.
- 같은 기종의 자동차는 중복해서 선택할 수 없다.
- `기종-이름`의 짝은 `,`로 연결해서 입력한다.

- 기종마다 각각 다른 전진 조건을 가지고 있다. `0-10` 까지의
    - `BIKE`: 랜덤 숫자 생성 후 4 이하이면 1칸 전진
    - `CAR`:  랜덤 숫자 하나를 생성 후, 3일 때만 1칸 전진
    - `FERRARI`: 랜덤 숫자가 8 이상이면, 2칸 전진
    - `TAXI`: 랜덤 숫자가 짝수면 1칸 전진
    - `BUS`: 랜덤 숫자가 홀수이면 1칸 전진

- 사용자는 몇 번의 이동을 할 것인지를 입력할 수 있어야 한다.
- 자동차 경주 게임을 완료한 후 누가 우승했는지를 알려준다. 우승자는 한 명 이상일 수 있다.
- 우승자가 여러 명일 경우 쉼표(,)를 이용하여 구분한다.
- 사용자가 잘못된 값을 입력할 경우 `IllegalArgumentException`을 발생시키고, 다시 입력받는다

<br>

## 입출력 요구사항

#### 입력

- 경주할 자동차 기종과 이름을 `-`로 연결하여 입력
- 참여자 이름은 `,`로 연결하여 입력

```text
FERRARI-경연,TAXI-우테코,BUS-포비
```

- 시도할 횟수

```text
3
```

#### 출력

- 시도 횟수별 결과

```
경연 : --
우테코 : 
포비 : -

```

- 우승자 결과

```
최종 우승자 : 경연

```

- 공동 우승자 결과

```
최종 우승자 : 경연, 포비

```

## 프로그래밍 요구 사항

- [ ] 메서드 라인은 `10줄`을 넘기지 않는다
- [ ] 스트림 체이닝은 3단계를 넘어가지 않도록 구현한다. 3줄이 넘으면 리팩터링을 고려한다

허용 범위 예시

```java
return products.stream()
    .

filter(this::method1)
    .

map(this::method2)
    .

toList();
```

- [ ] 가능한 ArrayList<> 가 아닌 다른 자료 구조를 활용하여 구현한다
- [ ] 3항 연산자를 쓰지 않는다
- [ ] `camp.nextstep.edu.missionutils` 에서 제공한 api 는 직접 구현하여 사용한다
- [ ] 정규식을 사용하지 않고 검증을 구현해본다

## 환경 요구 사항

- [ ] 이어폰이나 귀마개를 끼지 않고 (소음을 차단하지 않고) 진행한다
- [ ] 유동 인구가 있는 외부 공간에서 진행한다
